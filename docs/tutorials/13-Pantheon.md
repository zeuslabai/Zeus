# Pantheon — Multi-Agent Orchestration

Pantheon is Zeus's multi-agent collaboration system. Create missions, assemble agent teams, decompose goals into tasks, and monitor progress in real-time through War Room chat.

## Concepts

| Concept | Description |
|---------|-------------|
| **Mission** | A high-level goal ("Build a REST API") |
| **Team** | Agents assembled to work the mission |
| **Tasks** | Individual work items decomposed from the goal |
| **War Room** | Chat room for mission communication |
| **Activity Feed** | Real-time log of agent actions |
| **Artifacts** | Files and outputs generated by the mission |

## Creating a Mission

### Via API

```bash
curl -X POST http://localhost:3001/v1/pantheon/missions \
  -H "Content-Type: application/json" \
  -d '{
    "goal": "Build a user authentication API with JWT tokens",
    "constraints": {
      "budget_tokens": 50000,
      "timeout_seconds": 600,
      "max_agents": 4,
      "require_review": true
    }
  }'
```

### Via WebSocket

```json
{"type": "pantheon_mission", "goal": "Build a REST API", "constraints": {"max_agents": 3}}
```

### What Happens

1. Zeus decomposes the goal into tasks using LLM planning (Prometheus)
2. Team assembled from available agents based on capabilities
3. Tasks assigned to agents, executed in dependency order
4. Progress streamed to War Room and SSE/WebSocket clients
5. Results collected, artifacts persisted, mission completes

## Mission Lifecycle

```
Created → Assembling → Executing → Reviewing → Complete
                          │                       │
                          ├── Paused → Executing   │
                          └── Failed ←─────────────┘
                                ↑
                          Cancelled (from any state)
```

## Monitoring Missions

### List Missions

```bash
curl "http://localhost:3001/v1/pantheon/missions?limit=20" | jq
```

### Mission Detail

```bash
curl http://localhost:3001/v1/pantheon/missions/<id> | jq
```

### Real-Time Events (SSE)

```bash
curl -N http://localhost:3001/v1/pantheon/missions/<id>/events
```

Events include:
- `pantheon_team_assembled` — Team built
- `pantheon_task_assigned` — Task given to agent
- `pantheon_agent_activity` — Agent working (tool calls, etc.)
- `pantheon_task_completed` — Task done
- `pantheon_mission_progress` — Progress percentage update
- `pantheon_mission_complete` — Mission finished

### Activity Feed

```bash
curl http://localhost:3001/v1/pantheon/missions/<id>/activity | jq
```

## Intervention

Control running missions:

```bash
# Pause
curl -X POST http://localhost:3001/v1/pantheon/missions/<id>/intervene \
  -d '{"action":"pause"}'

# Resume
curl -X POST http://localhost:3001/v1/pantheon/missions/<id>/intervene \
  -d '{"action":"resume"}'

# Cancel
curl -X POST http://localhost:3001/v1/pantheon/missions/<id>/intervene \
  -d '{"action":"cancel"}'

# Redirect (give new direction)
curl -X POST http://localhost:3001/v1/pantheon/missions/<id>/intervene \
  -d '{"action":"redirect","message":"Focus on the login endpoint first"}'
```

## War Room Chat

Every mission gets a chat room. You can also create standalone rooms.

### Create a Room

```bash
curl -X POST http://localhost:3001/v1/pantheon/rooms \
  -H "Content-Type: application/json" \
  -d '{"name":"general","room_type":"public"}'
```

Room types: `public` (anyone can join), `private` (invite only).

### Join / Leave

```bash
curl -X POST http://localhost:3001/v1/pantheon/rooms/<id>/join \
  -d '{"agent_id":"user-1","agent_name":"Alice"}'

curl -X POST http://localhost:3001/v1/pantheon/rooms/<id>/leave \
  -d '{"agent_id":"user-1"}'
```

### Send Messages

```bash
curl -X POST http://localhost:3001/v1/pantheon/rooms/<id>/messages \
  -H "Content-Type: application/json" \
  -d '{
    "sender_id":"user-1",
    "sender_name":"Alice",
    "content":"Hello team!",
    "message_type":"chat"
  }'
```

Message types: `chat`, `system`, `tool_call`, `task_update`.

### Reactions

```bash
# Add reaction
curl -X POST http://localhost:3001/v1/pantheon/rooms/<id>/messages/<msg_id>/reactions \
  -d '{"user_id":"user-1","emoji":"👍"}'

# Remove reaction
curl -X DELETE http://localhost:3001/v1/pantheon/rooms/<id>/messages/<msg_id>/reactions \
  -d '{"user_id":"user-1","emoji":"👍"}'
```


## Recovery

Pantheon automatically recovers from failures:
- **Stale missions** — Missions stuck in Executing/Assembling for >5 minutes are marked Failed on gateway restart
- **Mission timeouts** — Checked every 60 seconds, auto-fails missions exceeding their timeout
- **Spawn failures** — Up to 3 retries per agent task, with automatic replacement
- **Stale agents** — Agents with expired heartbeats (>10 min) marked Offline
